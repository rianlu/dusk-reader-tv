# 暮阅 TV 阅读页全屏重构设计

**日期**: 2026-04-30
**状态**: 首版已实现（2026-05-01）
**范围**: `ReaderScreen` / `ReaderViewModel` / 阅读设置与分页逻辑

---

## 1. 目标

将当前阅读页从“双栏 spread 阅读”调整为更符合小说场景的“单页全屏阅读”。

本次重构需要同时解决以下问题：

1. 文本未铺满全屏，右半屏长期空白
2. 分页按固定 `1920x1080` 计算，导致不同 TV 分辨率下排版与加载异常
3. 阅读交互与遥控器心智不一致
4. 设置项虽有部分状态，但没有真正形成稳定可用的阅读体验

---

## 2. 设计结论

### 2.1 阅读模型

阅读页统一改为 **单页全屏文本**：

- 每一页只渲染一段完整可见的正文区域
- 不再使用左右双栏排版
- 不再保留当前 `ReaderSpread(leftText, rightText)` 的阅读表达
- 默认阅读行为是“沉浸模式”，正文铺满屏幕安全区

### 2.2 翻页模型

统一采用 **遥控器翻页**：

- `DPAD_LEFT`：上一页
- `DPAD_RIGHT`：下一页
- `DPAD_CENTER` / `ENTER`：呼出控制层
- `BACK`：按状态关闭目录/设置/控制层；沉浸态下直接退出阅读

本次不把“上下滚动模式”作为保留能力。原因是它与 TV 遥控器小说阅读的主要心智冲突，也会让分页和进度状态复杂化。

### 2.3 设置模型

保留并完善以下设置：

- 字体大小
- 行间距
- 段间距
- 阅读背景主题

背景主题至少提供 4 组：

1. 深色黑底
2. 护眼米色
3. 浅灰纸面
4. 墨绿色夜读

设置采用 **草稿态 + 确认应用 + 取消回退** 模型：

- 打开设置时读取当前值到 draft
- 调整设置不会立刻污染正式状态
- 点击确认后统一应用并重新分页
- 点击取消后恢复原状态

### 2.4 加载与分页模型

分页必须基于 **实际渲染区域尺寸** 动态计算：

- 由 Compose 实际测量正文容器宽高
- 将该尺寸传给 `ReaderViewModel.performPaging`
- 禁止继续写死 `1920x1080`

重新分页触发场景：

- 首次打开书籍
- 切换字体大小
- 切换行间距
- 切换段间距
- 切换背景主题后如果只影响颜色，不强制重新分页

分页与跳转规则：

- 保持当前阅读进度比例
- 目录跳转后必须稳定落到目标章节附近第一页
- 向前/向后翻到窗口边界时，继续加载前后窗口
- 清空页缓存后必须保证一定会重新触发分页

当前实现状态：

- `已实现` 阅读模型已切换为单页全屏正文
- `已实现` 使用 `HorizontalPager` 承载单页，而非双栏 spread
- `已实现` `DPAD_LEFT` / `DPAD_RIGHT` 作为主翻页方式
- `已实现` `DPAD_CENTER` 呼出控制层，`BACK` 按层级回退
- `已实现` 字号 / 行距 / 段距 / 4 组主题的草稿态设置
- `已实现` 正文分页基于实际正文容器尺寸触发
- `待继续` 阅读设置持久化到全局偏好
- `待继续` TV 模拟器上的长时稳定性与多分辨率回归

## 3. 页面结构

### 3.1 沉浸阅读态

正文区域：

- 占据屏幕主阅读区
- 左右留安全边距，但正文必须视觉上铺满整页，而不是只占半屏
- 不显示显式“上一页 / 下一页”大按钮

### 3.2 控制层

按 `DPAD_CENTER` 呼出控制层，结构如下：

- 顶部栏：书名、章节名、阅读进度、时间
- 底部栏：目录、设置、退出

控制层不再承担主要翻页职责，只承担辅助操作入口。翻页交给遥控器方向键完成。

### 3.3 目录层

目录层保持左侧抽屉形式：

- 展示章节列表
- 确认后跳转到章节
- 跳转完成后回到阅读态

### 3.4 设置层

设置层保持右侧抽屉形式：

- 字体大小
- 行间距
- 段间距
- 背景主题
- 确认
- 取消

## 4. 技术调整

### 4.1 UI 层

`ReaderScreen.kt` 需要完成以下调整：

- 将双栏 `Row + left/right text` 改成单页正文组件
- 使用 `BoxWithConstraints` 或 `onSizeChanged` 获取真实正文区域尺寸
- 把正文尺寸传给分页逻辑
- 移除“竖向模式”相关分支与设置项
- 移除控制层中的“上一页 / 下一页”大按钮
- 保留目录、设置、退出

### 4.2 ViewModel 层

`ReaderViewModel.kt` 需要完成以下调整：

- 将 `ReaderSpread` 页模型改成单页文本页模型
- 分页逻辑按单页长度推进，而不是按双页 spread 消耗
- 修正重新分页后的落点恢复
- 保持目录跳转、窗口前后加载、进度保存逻辑稳定

### 4.3 测试

测试覆盖至少包括：

1. 段间距变化会改变格式化文本
2. 单页分页结果不会再生成左右双栏结构
3. 重新分页时会保留阅读进度比例
4. 章节跳转后会触发新的分页请求

另外需要补一轮 TV 模拟器验证：

1. 首页打开书籍进入阅读页
2. 左右键翻页
3. `CENTER` 打开控制层
4. 设置字体大小后正文密度变化
5. 切换背景主题后颜色生效
6. 目录跳转后页面刷新正常
7. `BACK` / 退出可返回详情或上一级

---

## 5. 非目标

本次不做以下内容：

- EPUB 专属排版增强
- 阅读动画美化
- 字体族切换
- 亮度系统级联动
- 阅读页外的首页/传书页新需求扩展

---

## 6. 验收标准

满足以下条件才算完成：

1. TV 模拟器上正文视觉上为单页全屏阅读，不再出现右半屏长期空白
2. 左右键可稳定翻页
3. 字体大小和背景主题设置真实可用
4. 不同分辨率下分页不再依赖固定 `1920x1080`
5. 打开、跳章、重新分页、退出阅读都没有明显卡死或状态错乱
